MERGE PROMPT
--------------------------------------------------------------------------------

You are merging the results of 3 sequential M3 vision chunks into a single
coherent dataset. No inference, no re-analysis. Pure structural merge plus
sanity checks.

Steps:

  1. Load all 3 chunk JSONs in order (chunk_01, chunk_02, chunk_03).
  2. Concatenate the "queue" arrays in chunk order. The result must contain
     exactly 52 items (25 + 25 + 2) with no duplicates by dedupe_hash.
  3. For every item, verify:
       - venue_name is non-empty
       - resolved_image_url is non-empty
       - source_url is non-empty
       - "vision" object exists
       - vision.model_used == "MiniMax-M3"
       - either vision.venue_observed is filled OR vision.skipped == true
         with a valid skip_reason
     If any check fails, do NOT silently drop the item. Put it in a
     "validation_failures" list in the merged output with the reason.
  4. Build aggregate counts:
       - items_attempted = 52
       - items_succeeded = count where vision.skipped is not true
       - items_skipped = count where vision.skipped == true
       - skipped_reasons_summary = Counter of skip_reason values
       - per_venue_counts = Counter of venue_name
       - per_venue_succeeded = Counter of venue_name for non-skipped only
  5. Detect any item that appears in the sanitized queue but NOT in any
     chunk output. Report them as "missing_from_chunks" with the URL and
     venue. These are hard failures — flag, do not invent.
  6. Detect any item that appears in MORE than one chunk (by dedupe_hash).
     Report as "duplicates_across_chunks". Should be zero.
