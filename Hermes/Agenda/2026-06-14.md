### Señales de "web vieja" (filtros)
- Diseño 2010-2015, sin mobile
- Sin SSL ("No seguro")
- Sin WhatsApp clickeable
- Sin Google Reviews / Maps
- Última actualización > 2 años
- `.com.ar` con frames / flash / HTML estático
- No tiene sitemap.xml / robots.txt
- HTML < 80 KB (señal barata de stack viejo)
- Facebook/IG activo pero web muerta (migración)

### Filtros de viabilidad comercial
- ✅ Teléfono/WhatsApp público
- ✅ Google Maps listing
- ✅ Rubro que ya compra marketing
- ✅ Dueño visible (no corporativo)
- ❌ Evitar: gobierno, ONGs, familiares sin decisión de compra

### Deploy propuesto (3 fases)

**Fase 1 — Scraper (Día 1)**
- Script Python con Serper → 200 candidatos por rubro/ciudad
- Output: CSV con `nombre, web, telefono, direccion, rating, ciudad, rubro`

**Fase 2 — Score automático (Día 1-2)**
- Para cada candidato:
  ```bash
  curl -sI https://sitio.com  # SSL, server, last-modified
  curl -s https://sitio.com | grep -i "generator\|wordpress\|joomla"
  curl -s https://sitio.com | wc -c   # tamaño HTML
  ```
- Whois → año de registro del dominio
- Score 0-100 basado en:
  - Sin SSL: -30
  - HTML < 50KB: -25
  - Sin viewport mobile: -20
  - Sin sitemap.xml: -15
  - Stack viejo (FrontPage, Joomla 1.5, WordPress < 4): -10
  - Dominio registrado > 8 años: +10 (maduro, duele más)
- Output: tabla en Supabase con score + teléfono limpio

**Fase 3 — Outreach manual (Día 2-3)**
- Top 50 con score alto + WA público
- Mockup personalizado de home actual vs renovada (estilo Farias mockup)
- Outreach manual uno a uno (NO daemon hasta validar)
- 5-10/día máximo
