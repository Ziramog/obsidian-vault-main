## Problemas encontrados y resueltos

### 1. Bug timezone `isBusinessHours()` — CRÍTICO
- **Problema**: El daemon usaba `(utcHour - 3) % 24` para calcular hora Argentina. Esto daba Hour: 6 cuando eran las 09:00 (UTC 12 = Argentina 9, pero `(12-3)%24 = 9` no era el bug real... el bug era la fecha UTC al crear `argDate`). Resultado: siempre "Outside schedule" aunque fuera horario comercial.
- **Fix**: Reescrito usando `Intl.DateTimeFormat` con `timeZone: 'America/Argentina/Buenos_Aires'` — cálculo correcto sin bugs de wrap-around.
- **Archivo**: `/home/hermes/workspace/projects/outreach-connect-daemon/daemon.js` líneas 47-58

### 2. Bug EACCES state.json — CRÍTICO
- **Problema**: PM2 root lanzaba el daemon, pero state.json era de hermes. Cada ciclo crasheaba con EACCES.
- **Fix**: `sudo chmod 666 state.json` + configurado PM2 hermes para gestionar el proceso correctamente.
- **Archivo**: `/home/hermes/workspace/projects/outreach-connect-daemon/state.json` (permisos 666)

### 3. Bug Baileys 401 loop — CRÍTICO
- **Problema**: Cuando WhatsApp se desconectaba con código 401, `connect()` no resolvía/rechazaba la promesa, dejando el daemon colgado indefinidamente. Cada restart del daemon repetía el mismo error.
- **Fix**: En `baileys-relay.js`, cuando `isLoggedOut=true` (401), ahora llama recursivamente `connect()` para pedir un QR fresco en vez de retornar sin resolver.
- **Archivo**: `/home/hermes/workspace/projects/outreach-connect-daemon/baileys-relay.js`

### 4. Bug ecosystem.config.js — CONFIGURACIÓN
- **Problema**: El archivo tenía `cwd` apuntando a `/home/hermes/workspace/autonomous-daemon` y `name: 'autonomous-daemon'` — procesos fantasma.
- **Fix**: Corregido `cwd`, `name` y agregado `exec_mode: 'fork'`.
- **Archivo**: `/home/hermes/workspace/projects/outreach-connect-daemon/ecosystem.config.js`
