### VPS — OOM Crisis Resolved

**Problema:** VPS se caía sin motivo aparente. Root cause: **OOM Killer** — kernel Linux mata procesos cuando se queda sin RAM. `dmesg` showed `node invoked oom-killer` — proceso node consumió 6GB+ RSS. VPS tiene 8GB RAM, swap estaba desactivado.

**Solución aplicada:**
1. Swap 2GB creado y activado: `/swapfile`
2. Swap añadido a `/etc/fstab` (sobrevive reinicios)
3. outreach-api reiniciado: bajó de 123MB → 39MB (memory leak limpio)
4. outreach-api + outreach-daemon: siguen online ✅

**Post-fix memory snapshot:**
- RAM: ~1.5GB usado / 8GB total
- Swap: 2GB disponible
- Chrome headless: ~236MB (principal consumidor de RAM)

Con swap activo, VPS no debería caerse más por OOM.
