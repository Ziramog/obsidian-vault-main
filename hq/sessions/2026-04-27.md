## VPS Session — Late Afternoon (this session)

### Problem: VPS Crash — OOM Killer
- VPS se caía sin motivo aparente
- Root cause: **OOM Killer** — el kernel de Linux mata procesos cuando se queda sin RAM
- Detection: `dmesg` showed `node invoked oom-killer` — proceso node consumió 6GB+ RSS
- VPS tiene 8GB RAM, swap desactivado — cualquier pico de Chrome o node mataba el server

**Fixes applied:**
1. Swap 2GB creado y activado: `/swapfile`
2. Swap añadido a `/etc/fstab` (sobrevive reinicios)
3. outreach-api reiniciado (bajó de 123MB → 39MB, liberó memory leaks)
4. outreach-api + outreach-daemon: siguen online ✅

**Memory snapshot post-fix:**
- RAM: ~1.5GB usado / 8GB total
- Swap: 2GB disponible
- Con swap activo, VPS no debería caerse más por OOM
